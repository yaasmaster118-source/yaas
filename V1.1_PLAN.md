# YAAS 1.1

YAAS 1.1 is developed separately from the live 1.0 release.

## Release workflow

- `main`: live production version
- `codex/v1.1`: private development work
- Render deploys only `main`
- Do not merge or push 1.1 to `main` until the user says: `web siteye yansit`

## Product rules

1. New accounts start with no servers, channels, posts, or fake members.
2. A user can create a server or join one through an invite.
3. Non-members cannot see a server's channels, messages, members, or voice rooms.
4. Every server owns separate:
   - members
   - roles
   - permissions
   - text channels
   - voice channels
   - messages
   - invites
5. The server creator receives the Owner role.
6. Built-in role templates:
   - Owner
   - Admin
   - Moderator
   - Member
7. Server owners can create custom roles and assign permissions.
8. Permissions are checked by the backend, not only hidden in the interface.

## Permission model

- `server.view`
- `server.manage`
- `members.view`
- `members.manage`
- `roles.manage`
- `channels.manage`
- `channel.view`
- `messages.send`
- `messages.manage`
- `voice.join`
- `voice.speak`
- `voice.mute_members`
- `invites.create`

## Mobile

- Bottom navigation on phones
- Slide-over server and member panels
- Touch-friendly channel controls
- Mobile voice-room controls
- Responsive role and permission editor

## Required production services

- PostgreSQL database
- Persistent media storage
- Secure server-side sessions
- TURN server for reliable voice connections
